
| Property | Value |

### Tag Filter Bar

```
[kubernetes] [terraform] [SRE] [python]    × Clear filters
```

| Property | Value |
|----------|-------|
| Layout | `display: flex; flex-wrap: wrap; gap: 0.375rem; align-items: center` |
| Chip shape | `border-radius: 9999px` (pill) — matches `.post-card__tag` exactly |
| Chip padding | `0.125rem 0.5rem` (2px 8px) — matches `.post-card__tag` exactly |
| Chip font | 12px / 500 |
| Chip element | `<button type="button">` (not `<li>`) — must be keyboard-navigable |
| `aria-pressed` | `"true"` when active, `"false"` when inactive |
| Cursor | `cursor: pointer` |

**"Clear filters" control:**
- Appears inline at end of tag list when ≥1 tag is active OR search query is non-empty
- Label: `"× Clear filters"` (or `"Clear"` if space is tight)
- Style: plain link — `font-size: 0.875rem; color: var(--color-accent); background: none; border: none; cursor: pointer; text-decoration: underline on hover`
- Clears both active tags AND search input simultaneously

## Interaction Spec

### Filter Logic
- Tag filter + search work additively (AND across both dimensions)
- Tag logic: AND (post must match ALL active tags)
- Search: Fuse.js with `ignoreLocation: true`; fields: `title`, `tags`, `description`
- Filtering triggers: every keystroke for search, every chip click for tags

### Keyboard & Accessibility
- Search input: standard `<input type="search">` — native keyboard support
- Tag chips: `<button type="button">` elements — tab-focusable, activatable with Enter/Space
- `aria-pressed` on each chip button reflects toggle state
- "Clear filters" button: must be keyboard-reachable (inline in DOM after tag chips)
- Post list: existing `<article>` elements remain in DOM (use `display: none` to hide filtered cards — not removal — to preserve order and avoid ARIA live region complexity)
- Add `aria-live="polite"` region showing filtered count or empty state for screen readers

### Data Serialization
- Post data embedded at build time in a `<script type="application/json" id="post-data">` block in `posts/index.astro`
- Fields per post: `id` (slug), `title`, `tags` (array), `description` (excerpt or empty string)
- Fuse.js reads this on `DOMContentLoaded`

### Post Card Updates (`PostCard.astro`)
- `<article>` element: add `data-tags="{comma-separated-tags}"` attribute
- Each `<li class="post-card__tag">`: change to `<button type="button" class="post-card__tag" data-tag="{tag}">` for clickability
- Active state class: `post-card__tag--active` triggers accent fill styles
- Tags in post cards trigger the same filter toggle as the filter bar chips

## Copywriting Contract

| Element | Copy |
|---------|------|
| Search input placeholder | `Search posts…` |
| Search `aria-label` | `Search posts` |
| Tag filter bar `aria-label` | `Filter by tag` |
| Empty state — search only | `No posts match "{query}".` |
| Empty state — tags only (1 tag) | `No posts tagged "{tag}".` |
| Empty state — tags only (2+ tags) | `No posts match all selected tags.` |
| Empty state — combined | `No posts match "{query}" with the selected tags.` |
| Clear filters button label | `× Clear filters` |
| Clear filters `aria-label` | `Clear all filters` |
| Post count (existing) | `{N} post` / `{N} posts` — already present, update to reflect filtered count |

**Empty state rules:**
- Always plain text — no illustration, no secondary CTA
- Quotes around query string: straight quotes `"` (not curly)
- When count drops to 0: replace `.post-list` content with empty state paragraph; do not hide the controls
