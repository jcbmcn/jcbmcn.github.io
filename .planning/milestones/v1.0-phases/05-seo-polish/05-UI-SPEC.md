### Dark Mode Toggle Script

Embedded as an **inline `<script>` block in `BaseLayout.astro` `<head>` (before CSS paint):**

```
1. Read localStorage.getItem('theme')
2. If value is 'dark' or 'light' → apply as data-theme on <html>
3. If no value → check window.matchMedia('(prefers-color-scheme: dark)')
   - true → set data-theme="dark"
   - false → set data-theme="light"
```

**Toggle click behavior (separate `<script>` in `SiteHeader.astro` or inline in page):**
```
1. Read current data-theme from <html>
2. Toggle to opposite value
3. Write new value to localStorage under key "theme"
4. Update button aria-label to match new state
5. Swap visible icon (sun ↔ moon)
```

**Source:** Phase 5 CONTEXT.md (D-10, D-11, D-12, D-13)

## Interaction Spec

### Dark Mode Toggle

| State | Behavior |
|-------|----------|
| First visit — OS dark | `data-theme="dark"` applied before paint; moon icon replaced with sun; `aria-label="Switch to light mode"` |
| First visit — OS light | `data-theme="light"` applied; moon icon shown; `aria-label="Switch to dark mode"` |
| Click toggle | Flip `data-theme` on `<html>`, persist to `localStorage("theme")`, swap icon, update `aria-label` |
| Revisit with saved preference | Anti-flash inline script reads localStorage and applies `data-theme` before CSS renders |
| No flash | Guaranteed by inline script placement in `<head>` before any `<link>` or `<style>` |

### Theme Persistence
- Key: `"theme"`
- Values: `"light"` | `"dark"`
- Storage: `localStorage`
- Scope: entire site (applies on every page via `BaseLayout.astro`)

### SiteHeader Stickiness
- `position: sticky; top: 0; z-index: 10`
- Header remains visible during scroll on all pages
- No JavaScript required — pure CSS sticky
